# 如何添加海报图片到 GitHub

## 问题：图片无法显示

原因：图片文件还没有上传到 GitHub 仓库。

---

## 🎯 解决方案（3 种方式）

### 方式 1：从聊天记录保存图片 ⭐ 推荐

1. **找到你发送给我的那张海报图片**
   - 在聊天记录中找到图片
   - 右键点击图片
   - 选择"另存为"

2. **重命名图片**
   ```
   保存为: framedetective-banner.jpg
   ```

3. **移动到项目文件夹**
   ```
   复制到: d:\sammodel\assets\framedetective-banner.jpg
   ```

4. **提交到 GitHub**
   ```bash
   cd d:\sammodel
   git add assets/framedetective-banner.jpg
   git commit -m "Add project banner image"
   git push origin main
   ```

---

### 方式 2：使用在线图片链接（临时方案）

如果你的图片已经在网上：

1. **上传图片到图床**
   - imgur.com
   - imgbb.com
   - sm.ms

2. **获取直链**
   - 复制图片的直接链接（以 .jpg 或 .png 结尾）

3. **修改 README.md**
   ```markdown
   <img src="https://你的图片链接.jpg" alt="FrameDetective Banner" width="100%">
   ```

---

### 方式 3：使用 GitHub Issues（推荐）

1. **在 GitHub 创建一个 Issue**
   - 访问: https://github.com/Health-525/FrameDetective/issues/new

2. **拖拽图片到 Issue 编辑框**
   - GitHub 会自动上传图片
   - 复制生成的图片链接

3. **在 README.md 中使用这个链接**
   ```markdown
   <img src="复制的链接" alt="FrameDetective Banner" width="100%">
   ```

4. **关闭或删除这个 Issue**

## 🎨 图片规格建议

- **格式**: JPG 或 PNG
- **尺寸**: 1920x600 或 1920x1080 (16:9 比例)
- **文件大小**: < 500KB (使用 TinyPNG.com 压缩)
- **内容**: 你之前分享的那张 FrameDetective 海报

## 🆘 常见问题

### Q: 图片上传后还是不显示？

A: 可能需要几分钟让 GitHub 缓存更新。尝试：
- 刷新页面（Ctrl + F5 强制刷新）
- 清除浏览器缓存
- 等待 5-10 分钟

### Q: 图片太大无法上传？

A: GitHub 单文件限制 25MB，但建议压缩到 < 500KB：
- 使用 TinyPNG.com 在线压缩
- 使用 PhotoShop/GIMP 导出为 Web 格式

### Q: 我的图片在哪里？

A: 检查这些位置：
- 聊天记录中的原图
- 电脑下载文件夹
- 或重新生成一张海报
